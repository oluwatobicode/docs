The performance API will expose transaction traces and web vitals data for your project. The paths below are the **planned** routes; the request/response shapes describe the intended design.

<Warning>
  Performance monitoring is **not yet implemented**. These endpoints are not live — don't call them in production yet. This page documents the planned API so integrators can prepare; shapes may change before release. Error tracking, alerts, and billing are fully available today.
</Warning>

## Authentication

Both performance endpoints require an active session. Requests without a valid session cookie return `401 Unauthorized`.

---

## List Transactions

Retrieve transaction records collected for a project. Each transaction represents a traced operation — a page load, an API call, or a background job — with a name, trace ID, status, and duration.

```
GET /api/v1/projects/:projectId/performance/transactions
```

### Path Parameters

<ParamField path="projectId" type="string" required>
  The ID of the project whose transactions you want to retrieve.
</ParamField>

### Example Request

```bash
curl https://your-domain.com/api/v1/projects/clxk2m9ab0000qw8e3f7t1pz2/performance/transactions \
  -H 'Cookie: session=YOUR_SESSION_COOKIE'
```

---

## Get Web Vitals

Retrieve web vitals measurements for a project. Web vitals reflect real-user experience across key metrics including Largest Contentful Paint (LCP), Cumulative Layout Shift (CLS), Time to First Byte (TTFB), and First Contentful Paint (FCP).

```
GET /api/v1/projects/:projectId/performance/vitals
```

### Path Parameters

<ParamField path="projectId" type="string" required>
  The ID of the project whose web vitals you want to retrieve.
</ParamField>

### Example Request

```bash
curl https://your-domain.com/api/v1/projects/clxk2m9ab0000qw8e3f7t1pz2/performance/vitals \
  -H 'Cookie: session=YOUR_SESSION_COOKIE'
```

## Response Codes

| Status | Meaning                                          |
|--------|--------------------------------------------------|
| `200`  | Request succeeded.                               |
| `401`  | Not authenticated — no valid session.            |
